2. NEW FEATURES ADDED AFTER PHASES
2.1 Daily Notes (at /today)
A daily scratchpad on the Tour Day View. Write notes, bullet points, numbered lists, and checkboxes while on the road. Auto-saves every 2 seconds. Organized by date and by tour.

Quarter-panel on the right side of /today
Supports: plain text, bullet points (- or *), numbered lists (1. 2. 3.), checkboxes ([ ] and [x])
Auto-saves 2 seconds after you stop typing — shows a subtle "Saved" indicator
Visibility selector per note: Only Me / Band Admin Can See / Whole Band Can See — default: Only Me
Active tour auto-detected from today's show or most recent active tour
Past Notes button opens modal with two tabs: By Date and By Tour
By Date: scrollable list of dates with notes, click to read that day's note
By Tour: dropdown of all tours showing notes written during that tour period
Notes from other authors shown in different color with author name
Members and Band Admins see notes shared with their role — cannot see agent-only notes
DATABASE: daily_notes table with user_id, act_id, tour_id, note_date, content, visibility, RLS policies.
2.2 Expense Tracking (at /financials)
Expenses live only in /financials — not on /today. Each expense is tied to a tour (required) and optionally a specific show. Feeds net income calculation across the platform.

Expense entry via "+ Add Expense" button on the Expenses tab in /financials
Each expense: Date, Tour (required), Show/Booking (optional), Category, Amount, Notes
Default categories: Gas / Mileage, Hotel / Lodging, Band Member Payments, Food / Meals, Equipment, Other
Per-show expenses: see what each show night cost
Per-tour rollup: total expenses for the full run
Summary bar: total by category
Filter by tour, date range, and category
Edit and delete per row with confirmation
Net Income = Total Earned minus Total Expenses — all financial cards updated to reflect real expense data
Tour detail page shows expense summary card with "+ Add Expense" button
DATABASE: expenses table with user_id, act_id, tour_id, booking_id, expense_date, category, amount, notes, RLS policies.

3. THE COMPLETE BOOKING LIFECYCLE
Stage 1 — Design the Tour
Create tour with name, band, date range (date picker)
Define target cities for the run
System shows routing context — city order, drive windows
Wed-Sat show windows highlighted as primary booking targets
Stage 2 — Discover Venues
Google Places search by city and state
Email is prioritized — booking contact email surfaced first
Venue added to both tour outreach pool AND main venue database simultaneously
Auto-Find Emails: scrapes venue websites for missing booking contact emails
Bulk import with downloadable CSV template
Genre filtering with multi-select (music_genres TEXT[] column)
Stage 3 — Initial Outreach
AI generates cold pitch draft for each venue in outreach pool
Subject: "[Band Name] — Booking Inquiry — [Venue], [City]"
Agent sends include: "Sent on behalf of [Band] by Camel Ranch Booking" signature
Send individually or bulk send
Status updates to Initial Contact Sent with timestamp
Logged in outreach tracker
Stage 4 — Follow-Up Automation
7-day timer starts at send
No reply after 7 days: red badge on venue card, notification created
Reply received: venue moves to In Conversation status, notification sent
All follow-up actions logged with timestamps
Stage 5 — Confirm the Booking
CONFIRM SHOW button on venue in outreach pool
Creates booking record with deal type, agreed amount, confirmed_by, details_pending flag
Calendar event created immediately — status Details Pending until logistics filled
Confirmation email sent to venue automatically — includes band, agent, show date, deal
Venue fills in logistics form — details push into booking record and calendar
Stage 6 — The Booking Record
All fields configurable per band admin
Stage 7 — Advance the Show
14 days before: advance reminder notification
Pending logistics flagged amber on Tour Day View
Stage 8 — Tour Day View (/today)
Agent/Band Admin: venue, drive time, load-in/soundcheck/showtime, hotel, meals, drinks, sound, deal/pay
Member: same operational details — NO deal type, NO pay amounts, NO financial data
Daily Notes panel
Stage 9 — Settle
Mark as Played button
Post-show form: actual pay, payment status, notes, rebook flag
Expense tracking: add tour/show expenses
Stage 10 — Show History & Financials
Every played show permanently archived at /history
Annual summary at /financials
Expense tracking feeds Net Income calculation

8. DATABASE MIGRATIONS — RUN THESE IN SUPABASE
8.1 Daily Notes Table
CREATE TABLE daily_notes (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  act_id uuid REFERENCES acts(id) ON DELETE SET NULL,
  tour_id uuid REFERENCES tours(id) ON DELETE SET NULL,
  note_date date NOT NULL,
  content text NOT NULL DEFAULT '',
  visibility text NOT NULL DEFAULT 'agent_only'
    CHECK (visibility IN ('agent_only','band_admin','all_members')),
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX daily_notes_user_date_idx ON daily_notes (user_id, note_date);
ALTER TABLE daily_notes ENABLE ROW LEVEL SECURITY;

8.2 Expenses Table
CREATE TABLE expenses (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  act_id uuid REFERENCES acts(id) ON DELETE SET NULL,
  tour_id uuid REFERENCES tours(id) ON DELETE SET NULL,
  booking_id uuid REFERENCES bookings(id) ON DELETE SET NULL,
  expense_date date NOT NULL,
  category text NOT NULL,
  amount numeric NOT NULL,
  notes text,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);
ALTER TABLE expenses ENABLE ROW LEVEL SECURITY;
